## Core Rules for Claude

- **Never invent brand decisions.** If something is not documented as Approved, treat it as Candidate or Experimental.
- **System A (Identity)** is minimal and timeless. It must work in one color and at small sizes. Do not add scenery, glows, or effects to identity assets.
- **System B (Marketing)** is cinematic and can be rich, but it must always support - never replace or overcrowd - the System A identity.
- Only move items to `15-VAULT/` if they have an explicit **Approved** status backed by a Decision Log entry.
- Always propose structural or brand changes via the RFC process in `19-RFC/`.

## How to Work Across Repositories

- Keep the Creative System as the single source of truth.
- When restructuring repos, follow the migration plans and maintain existing functionality.
- Document any new decisions in the Decision Log.

## How to Submit Changes

1. Create an RFC in `19-RFC/`
2. Update the Decision Log when a decision is made
3. Only promote items to Approved after proper review

## Uncertainty Protocol

If you are unsure about a brand decision, creative direction, or process:
- State the uncertainty clearly
- Reference the relevant document
- Propose the safest conservative approach
- Ask for human clarification before proceeding with irreversible changes

This system is designed to be followed, not improvised. When in doubt, refer back to the documents.
